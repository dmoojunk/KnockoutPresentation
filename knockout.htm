So... what is it?


  • Open Source JavaScript library

    - (MIT license) Source on GitHub with an active community

  • Data binding MVVM Pattern

    - a clear separation between domain data, view components and data to be displayed

  • Simplfies complex interactions

    - jQuery can quickly become an unreadable mess of interrelated event handlers.













MVVM - Model View ViewModel


  • Model

    - Server data

  • View

    - HTML markup

  • View Model

    - Client side model (javascript)













What does it do?


  • Automatic dependency tracking

    - updates the right parts of your UI whenever your data model changes.

  • Declarative bindings

    - a simple and obvious way to connect parts of your UI to your data model. You can construct a complex dynamic UIs easily using arbitrarily nested binding contexts.

  • Integrated Templating

    - using a native template engine although other templating engines can be used, such as jquery.tmpl, mustache etc.













But I love jQuery!

Other Stuff


  • SPA Applications

    - integrate with browser cache for a full offline experience

  • Easy to extend

    - hundreds of extensions available. Easy to write your own.

  • Integration with Server

    - ajax calls to get, then parse data automatically into a viewmodel. ability to flatten viewmodel for post back to the server.

  • Slower than jQuery for DOM manipulations

    - just dont make it bind and render a table of 1000 items.
